### `1.3.2`（2026-06-23，新增 `/dgm whoami` 诊断指令，纯附加零兼容性影响）

> **起因**：基岩玩家透过 Velocity proxy 进到后端 Paper 服务器，明明 `floodgate.jar` 两端都装了、`key.pem` 也对齐过，开选单却仍看到 Java 箱子 GUI 而非基岩原生 SimpleForm。`MiaoMenu#openSmartMenu` 的分流只依赖一句 `FloodgateApi.isFloodgatePlayer(uuid)`，但这道反射在 Velocity 后端 Floodgate 没正确接管玩家身份时就会回 false，且失败时不会印任何讯息，玩家／管理员无从排查。本版加一条玩家可自助呼叫的诊断指令，把所有环节印出来。

#### 指令／玩家路径
- **新增 `/dgm whoami` 子指令**（`commands/impl/WhoamiCommand.java`）：玩家专用，无权限门槛、仅显示呼叫者自己的信息。输出涵盖：
  - `floodgate` 与 `Geyser-Spigot` plugin 是否安装、版本字符串、启用状态
  - 玩家 UUID（提示「基岩玩家通常以 `00000000-0000-0000` 开头」）
  - smart-dispatch 判定（呼叫 `MiaoMenu#isBedrockPlayer` 取结果）
  - `FloodgateApi#isFloodgatePlayer(uuid)` 反射回传值
  - `FloodgateApi#getPlayer(uuid)` 取出的 Bedrock username、XUID、是否有 linkedPlayer
  - 反射失败时印出例外类型 + 讯息（unwrap `InvocationTargetException` 的 cause，避免空字符串）
  - 末尾依 smart-dispatch 结果印 verdict 行（基岩走 SimpleForm／Java 走箱子 GUI，后者附常见原因清单：Velocity key.pem 未对齐、auth-type 设定错误、linked Java 帐号绕过）
- **`MiaoMenu#isBedrockPlayer` 从 private 改 public**：让 `WhoamiCommand` 直接呼叫既有实作，不重新写一份反射逻辑。原方法行为完全不变，仅 visibility 调整。
- **`CommandManager`**：`registerCommands` 新增 `register("whoami", new WhoamiCommand(plugin))`；对应的 import 区块补上 `WhoamiCommand`。
- **`HelpCommand#defaultCommandOrder`**：加入 `"whoami"`（顺序：open / reload / import / lang / about / **whoami** / help），让 `/dgm help` 列表自动纳入新指令。

#### 语系文件
- **`lang/zh_TW.yml`**：新增 `descriptions.whoami`、`help.commands.whoami.{usage,detail,permission}`、整段 `whoami.*` 文案区块（header / floodgate-plugin / geyser-plugin / uuid / smart-dispatch / floodgate-api / is-floodgate-player / floodgate-username / floodgate-xuid / floodgate-linked / floodgate-error / verdict-bedrock / verdict-java / missing / enabled / disabled / true / false / null）。文案沿用 Lang `{0}` 占位符风格与项目既有 ✿ / ✦ icon 排版。
- **`lang/en.yml`**：同步加上完全对应的英文版键值。

#### 铁则达标
- 既有 `config.yml`（`config-version=16`）零异动
- 既有 `java_menus/*.yml`、`bedrock_menus/*.yml`、`plugin.yml` schema 零变更
- 既有指令、别名（`dgm` / `fluxmenu` / `mmf`）、权限 default 全保留
- 既有 50 个 source file 行为 0 变动，仅 1 个 public visibility 微调（不影响呼叫者）

### `1.3.1`（2026-06-20，多代理 6 轮健检循环，玩家操作零变动／配置文件零异动）

> 任务铁则：「**找问题 → 修问题 → 验证 ×2 通过为止**」且「**使用者操作起来要跟之前一样、配置文件不需要重新设定**」。三组独立子代理（找问题 / 处理 / 检验）平行运作，每轮换新代理避免上下文残留。共跑 6 轮、20 处修补、Discord webhook 全程同步进度（5 次推播）。连续两轮验证代理判 clean 才收尾。

#### Folia / 线程安全
- **`FoliaAdapter` + `FoliaFactory` 新增 `runForEntity(Plugin, Entity, Runnable)`**：立即执行版（与原 `runTaskLaterForEntity` 并存）。Bukkit 路径判 `isPrimaryThread` 直跑、否则 `runTask`；Folia 路径走 `entity.getScheduler().run`。
- **`PlayerAction#execute` 包进 `runForEntity`**：`player.performCommand(cmd)` 在 Folia 上必须在玩家所属 region 线程，原本被 Floodgate callback / fallback menu 触发时会在 Netty 线程丢 `IllegalStateException`。
- **`BedrockMenuManager#createFormResponseHandler` 整段包进 `runForEntity`**：Floodgate 表单 callback 由 Netty 线程触发，下游 `handleFormResponse` 会 `sendMessage` / 派发动作 / 开新选单，在 Folia 全部需排到玩家 region 线程；同步新增 `player.isOnline()` guard 避免表单开启到提交之间离线的例外。
- **`HotReloadManager` 锁防 disable race**：`lastConfigReloadTime` / `lastLangReloadTime` 加 `volatile`；`scheduleReload` 入口检 `!running || !plugin.isEnabled()`，避免 `onDisable` 期间 watcher 线程仍排程触发 `IllegalPluginAccessException`。
- **`Lang.messages` / `currentLanguage` 加 `volatile`**：UpdateChecker 的 HttpClient async 回调会呼叫 `Lang.get`；reload 主线程切换 `messages` 时非主线程可能读到 partially published reference。
- **`RateLimiter#allow` 改 `windows.compute(uuid, …)`**：原本 get→put 两步非原子，Folia 多 region 线程并发点击会被稳定绕过；改成 `ConcurrentHashMap.compute` 把「读旧→判超限→写新」收进原子段落。

#### 安全 / 防护
- **`InputValidator#isSafeCommandContent` 从白名单改黑名单**：原本 `[A-Za-z0-9_:\\-./%{} ]+` 会把 PAPI 展开后的 `&`、`§`、`[`、`]`、`、`、中文等正常字符全部误杀；改成黑名单只挡真正危险的 `CR/LF/TAB/NUL` 与 chain 字符串 `;; && || §§`（`§§` 防 `&&` 经 `PlaceholderUtils.parse` 转成 `§§` 后旁路）。`PlayerAction` 也补上同样的检查，与 `CmdAction` 对齐。
- **`JavaMenuListener#onClick` 防 shift-click 物品流失**：原本只用 `clickedInventory.getHolder() instanceof MenuHolder` 判断，玩家对自身物品栏 shift-click / NUMBER_KEY / SWAP_OFFHAND / COLLECT_TO_CURSOR 会把物品 transfer 进选单上层 inventory 造成永久遗失。改成先用 `event.getView().getTopInventory().getHolder()` 判定整个窗口，是选单就先 `setCancelled(true)`，再决定是否派发动作。
- **`ProxyManager#detectProxyType` 修跨服误判**：原本 `Class.forName("net.md_5.bungee.api.ChatColor")` 在 Paper 上永远 true（Paper 自带 shaded class），所有单机 Paper 服务器都被判定为 BungeeCord 后端 → 选单 `/server <name>` 动作 silently 失败。改为：(1) `settings.velocity-network: true` opt-in 走 Velocity；(2) 新增 `settings.bungeecord-network: true` opt-in；(3) fallback 读 `Bukkit.spigot().getSpigotConfig().getBoolean("settings.bungeecord")`；都判 false 即 `NONE`、fail-closed。
- **`ItemResolver#resolveBase64Head` 限缩 base64 段**：必须匹配 `[A-Fa-f0-9]{16,128}`（SHA-1 风格 hex），防 PAPI 拼接出 `../@evil.tld/x` 之类奇怪字符串造成意外的 URL fetch。

#### 指令 / 权限 / UX
- **`plugin.yml` 主指令 `dgeysermenu` 补 `permission: dgeysermenu.use`**：与既有 `permissions:` 区段 default true 对齐，一般玩家完全无感；只有当管理员在 LuckPerms 显式将该节点设为 deny 才会被挡下（旧版亦该挡）。
- **`CommandManager` tab-complete 按权限过滤**：新增 `SUBCOMMAND_PERMISSIONS` map，对 `args 1` 列子指令与 `args>1` 深层 tab 皆依 sender 权限过滤。一般玩家现在看不到 `reload` / `import` / `lang` 等管理子指令（仍可看到 `open` / `about` / `help`）。
- **`LangCommand` 写 `config.yml` 后补 `plugin.reloadConfig()`**：避免热重载与其他模块内存 cache 不一致。

#### DeluxeMenus 汇入
- **`DeluxeMenusImporter#convertMenu` 修 `open_command` 为 list 时 toString**：原本 `src.getString("open_command")` 对 YAML list 会回传 `"[a, b]"` 字符串塞进 `_imported_open_command` 溯源字段。改用 `src.get(...) instanceof List<?>` 分支写 stringList、否则 `toString()`。
- **`DeluxeMenusImporter#convertActions` prefix 对应表大幅扩充**：原本只认 `[console]` / `[openguimenu]`，其余原样保留 → 被 `DefaultAction` 当玩家身份指令误触发。本版补：`[chat]/[broadcast]/[json]/[minimessage]/[actionbar]/[title]/[subtitle] → [message]`、`[refresh] → [close]`、`[connect] X → [player] server X`（走 ProxyManager 跨服）、`[sound] X → [cmd] playsound X %player_name%`（用 PAPI 注 selector）、`[commandevent] → [cmd]`、未知 prefix 仍原样保留。

#### 其他
- **`MenuClockManager#playerHasClock` 显式查副手与盔甲槽**：改用 `getStorageContents()` + `getItemInOffHand()` + `getArmorContents()` 三段，避免 Paper API 变动或非 Paper 衍生 server 漏掉副手 / 盔甲槽。
- **`pom.xml` 版本 1.3 → 1.3.1**。

- 既有 `java_menus/*.yml`、`bedrock_menus/*.yml`、`lang/*.yml` 零 schema 异动
- 既有指令 / 别名（`dgm` / `fluxmenu` / `mmf`）/ 权限 default 全部保留

### `1.3`（2026-06-20，DeluxeMenus click 动作完整对齐 + 汇入双写基岩版 + 选单指令稽核 log）

> Avery 实地测试 1.2 汇入结果后回报：「之前在 dmenu 设定的左键 / 右键 / 中键分别执行不同指令、或单一通用指令，汇入后完全没动作」「基岩版本你没转换进去」「正常打指令会有 `issued server command:` log，但用箱子面板却没有」。覆查发现多层遗失：`JavaMenuListener` 只认 left / right、`DeluxeMenusImporter` 也只搬这两条链与单一 `java_menus/` 输出、`PlayerAction` 走 `player.performCommand()` 没印 chat-packet 那层 log。本版一次补齐。

#### 新功能
- **DeluxeMenus 汇入自动双写**：`DeluxeMenusImporter` 新增 `convertMenuToBedrock(Path)` 与 `extractFirstBedrockAction(...)`，每次汇入同时产出 `java_menus/<id>.yml`（原箱子 GUI）+ `bedrock_menus/<id>.yml`（Floodgate SimpleForm），基岩端玩家 `/dgm open <id>` 立即看到对应表单，无需手动补对应 YAML。
  - **转换策略**：items 依 slot 升幂排成按钮列表（表单没 slot 概念）；`display_name + lore` 合成 `text`（以 `\n` 串接）；material 走内建 `MATERIAL_TO_BEDROCK_ICON` vanilla 对照表（42 项），未命中时 icon 留空 → fallback 为纯文字按钮，不会炸。
  - **动作取舍**：SimpleForm 一颗按钮只能跑一个动作，从 `left_click_commands`（空则 fallback `click_commands`）取「第一个」有效动作；prefix 对应与 Java 端一致（`[player]/[message]/[close]/[cmd]/[openguimenu] X`）。多动作或 right/middle/shift_* 全部舍弃并在输出 YAML 顶部注释标出受影响的 slot，提醒手动审阅对应的 `java_menus/` 版本。
  - **`MenuEntry` record 扩字段**：新增 `bedrockOutputRelative`，向后兼容 3 参数呼叫（自动从 `java_menus/<safeId>.yml` 推回 `bedrock_menus/<safeId>.yml`）。
  - **`LegacyImportService.applyDeluxeMenus` 双写**：抽 `writeConverted(dst, yaml, conflict)` helper 共用 ADD/SKIP/OVERWRITE/RENAME 逻辑，避免两段 switch drift；`previewDeluxeMenus` 每个 menu 产 2 个 entry，token 涵盖两端路径。

#### 修正
- **`JavaMenu.MenuItem` 补齐四种 click 命令清单**：新增 `middleClickCommands` / `shiftLeftClickCommands` / `shiftRightClickCommands` / `clickCommands`（最后一个为 DeluxeMenus 风格的通用 catch-all）+ 对应 getters；构造子签名由 9 参数扩为 13 参数，内部唯一呼叫点 `loadItems` 同步更新。
- **`JavaMenu#loadItems` 读 4 个新 YAML 键**：`middle_click_commands` / `shift_left_click_commands` / `shift_right_click_commands` / `click_commands`。键名直接对齐 DeluxeMenus 官方字段，汇入后免再改名。
- **`JavaMenuListener#onClick` 改 ClickType 分流**：原本只用 `isLeftClick()` / `isRightClick()` 两条分支 → 会把 `SHIFT_LEFT` 误吃进 left、`SHIFT_RIGHT` 误吃进 right、`MIDDLE` 完全没处理。改用 `event.getClick()` switch + DeluxeMenus 风格 fallback 链：
  - `SHIFT_LEFT` → shift_left → left → click
  - `SHIFT_RIGHT` → shift_right → right → click
  - `MIDDLE` → middle → click
  - `LEFT` / `DOUBLE_CLICK` → left → click
  - `RIGHT` → right → click
  - 其他（`DROP` / `NUMBER_KEY` / `CONTROL_DROP` / `UNKNOWN`）→ click（catch-all）
- **`DeluxeMenusImporter#convertMenu` 搬全部 6 个 click 键**：抽出 `copyClickList` helper，把 `left/right/middle/shift_left/shift_right/click_commands` 一次处理；prefix 对应（`[console]→[cmd]`、`[openguimenu] X→[player] dgm open X`）照样套用到每条清单。
- **`PlayerAction` 补印 server command 稽核 log**：原本 `player.performCommand(cmd)` / `ProxyManager.sendServerCommand(...)` 都不会印 Paper 那条 `<玩家> issued server command: /<cmd>`，导致选单触发的指令在主控台默默消失、难以追责。改为在分发前先用 `Bukkit.getLogger().info(...)` 印出与 Paper 原生**完全一致**的格式（含跨服 `server` 分流路径），既有开关／log handler 不变。
- **PlaceholderAPI 诊断可视化**：Avery 回报「PAPI 已装但占位符仍显示原文」，覆查发现程序码路径没问题（每个 parse 呼叫点都有传 plugin），多半是缺对应 expansion（例如 `%player_level%` 需要 `/papi ecloud download Player`）。新增两道诊断：(1) `MiaoMenu#logPlaceholderApiStatus` 在 onEnable 末段印一行「PAPI 侦测结果 + expansion 数量」，未装 / 0 expansion / 正常各有对应提示；(2) `PlaceholderUtils#parse` 内 `setPlaceholders` 例外从 FINE 抬升到 WARNING，expansion 内部炸掉时不再 silent。
- **基岩端「Java 独有动作」按钮明确提示**：DeluxeMenus 来源若某 item 只有 `right_click_commands` / `middle_click_commands` / `shift_left_click_commands` / `shift_right_click_commands` 而没有 `left_click_commands` / `click_commands`，汇入转成 Bedrock SimpleForm 时 `command` 会是空、原本基岩玩家点下去是 silent 无反应（误以为 lag）。改进：(1) `DeluxeMenusImporter#convertMenuToBedrock` 侦测到此情境，在 bedrock YAML 写 `unsupported_on_bedrock: true` 旗标；(2) `BedrockMenu.BedrockMenuItem` record 加 `unsupportedOnBedrock` 字段并由 loadMenuItems 读取；(3) `BedrockMenuManager#handleFormResponse` 在 `cmd` 空且旗标为 true 时，发送 `Lang.get("bedrock.unsupported-button")` 提示玩家「此动作仅 Java 版可执行」；(4) 新增 `bedrock.unsupported-button` lang key（双语）让服务器主可改文案。

### `1.2` 增订续（2026-06-20，DeluxeMenus 自动转换汇入 + 补翻译 + 作者标 Avery）

> Avery 看了 1.2 后三项加码：「DeluxeMenus 的汇入要支持」、「有些帮助页面根本没有翻译到中文」、「作者部分只要写我，原著 github 里写了」。读过真实 DeluxeMenus 文件夹后，把先前排除掉的 DeluxeMenus/dmenu 加回 import 候选，并做真正的 schema 转换（而非整批复制）。本段为 1.2 的后续增订，使用者既有指令／别名／权限／设定键零变动。

#### Folia / 线程安全
- **`FoliaAdapter` + `FoliaFactory` 新增 `runForEntity(Plugin, Entity, Runnable)`**：立即执行版（与原 `runTaskLaterForEntity` 并存）。Bukkit 路径判 `isPrimaryThread` 直跑、否则 `runTask`；Folia 路径走 `entity.getScheduler().run`。
- **`PlayerAction#execute` 包进 `runForEntity`**：`player.performCommand(cmd)` 在 Folia 上必须在玩家所属 region 线程，原本被 Floodgate callback / fallback menu 触发时会在 Netty 线程丢 `IllegalStateException`。
- **`BedrockMenuManager#createFormResponseHandler` 整段包进 `runForEntity`**：Floodgate 表单 callback 由 Netty 线程触发，下游 `handleFormResponse` 会 `sendMessage` / 派发动作 / 开新选单，在 Folia 全部需排到玩家 region 线程；同步新增 `player.isOnline()` guard 避免表单开启到提交之间离线的例外。
- **`HotReloadManager` 锁防 disable race**：`lastConfigReloadTime` / `lastLangReloadTime` 加 `volatile`；`scheduleReload` 入口检 `!running || !plugin.isEnabled()`，避免 `onDisable` 期间 watcher 线程仍排程触发 `IllegalPluginAccessException`。
- **`Lang.messages` / `currentLanguage` 加 `volatile`**：UpdateChecker 的 HttpClient async 回调会呼叫 `Lang.get`；reload 主线程切换 `messages` 时非主线程可能读到 partially published reference。
- **`RateLimiter#allow` 改 `windows.compute(uuid, …)`**：原本 get→put 两步非原子，Folia 多 region 线程并发点击会被稳定绕过；改成 `ConcurrentHashMap.compute` 把「读旧→判超限→写新」收进原子段落。

#### 安全 / 防护
- **`InputValidator#isSafeCommandContent` 从白名单改黑名单**：原本 `[A-Za-z0-9_:\\-./%{} ]+` 会把 PAPI 展开后的 `&`、`§`、`[`、`]`、`、`、中文等正常字符全部误杀；改成黑名单只挡真正危险的 `CR/LF/TAB/NUL` 与 chain 字符串 `;; && || §§`（`§§` 防 `&&` 经 `PlaceholderUtils.parse` 转成 `§§` 后旁路）。`PlayerAction` 也补上同样的检查，与 `CmdAction` 对齐。
- **`JavaMenuListener#onClick` 防 shift-click 物品流失**：原本只用 `clickedInventory.getHolder() instanceof MenuHolder` 判断，玩家对自身物品栏 shift-click / NUMBER_KEY / SWAP_OFFHAND / COLLECT_TO_CURSOR 会把物品 transfer 进选单上层 inventory 造成永久遗失。改成先用 `event.getView().getTopInventory().getHolder()` 判定整个窗口，是选单就先 `setCancelled(true)`，再决定是否派发动作。
- **`ProxyManager#detectProxyType` 修跨服误判**：原本 `Class.forName("net.md_5.bungee.api.ChatColor")` 在 Paper 上永远 true（Paper 自带 shaded class），所有单机 Paper 服务器都被判定为 BungeeCord 后端 → 选单 `/server <name>` 动作 silently 失败。改为：(1) `settings.velocity-network: true` opt-in 走 Velocity；(2) 新增 `settings.bungeecord-network: true` opt-in；(3) fallback 读 `Bukkit.spigot().getSpigotConfig().getBoolean("settings.bungeecord")`；都判 false 即 `NONE`、fail-closed。
- **`ItemResolver#resolveBase64Head` 限缩 base64 段**：必须匹配 `[A-Fa-f0-9]{16,128}`（SHA-1 风格 hex），防 PAPI 拼接出 `../@evil.tld/x` 之类奇怪字符串造成意外的 URL fetch。

#### 指令 / 权限 / UX

- **权限条件补旁路**：`requirement` 写 `type: permission` 但漏写 `permission:` 字段时，原本会放行 → 改为 fail-closed 直接拒绝，避免设定漏字导致玩家绕过权限。
- **`material` 容错**：`java_menus/*.yml` 与 `config.yml` 的 `fallback-material` 若使用者写成 YAML null（`~`）或空字符串，原本会丢 `IllegalArgumentException` 卡住 onEnable / 开选单；改为自动退回 `STONE`。
- **未知 requirement type 不再静默**：YAML 拼错 `type:`（例如 `permssion`）原本会被默默跳过、玩家以为设了条件其实全没套；改为印一次 warning（同 type 之后静音，不洗版）。
- **`ScoreboardManager` null 防护**：早期启动或 Folia 异常状态下 `getScoreboardManager()` 可能传 null；补上 null 检查，避免条件评估时 NPE。
- **热重载 watcher 线程清理**：`HotReloadManager.shutdown()` 改成「先关 `WatchService` → interrupt → `join(1000ms)`」顺序，确保插件 disable 时 watcher 线程能正确结束。
- **`SoundsClock` 防护**：插件 disable 期间若仍有开选单流程触发，包 try-catch 避免 `IllegalStateException` 中断整个流程。
- **`saveResourceIfNotExists` 不再静默**：jar 内找不到示例资源时原本直接吞例外，现在改 log warning。
- **`OpenCommand` tab 补全防护**：对 `args.length == 0` 与 `args[0] == null` 加边界检查。

#### 内存 / 资源
- **`RateLimiter` 加被动清理 + 主动移除**：每 60 秒 CAS 一次扫描过期条目（超过 5×window）、玩家离线时 `PlayerQuitEvent` 主动移除 UUID，长期运转服务器不再缓慢累积条目。
- **`MenuClockManager.playerHasClock` 改用 `getContents()`**：原本只扫主物品栏漏掉副手，玩家把选单时钟放副手后重连会被误判为「没时钟」而重复给予；改用 `getContents()` 扫全部字段。

#### PlaceholderAPI 一致性
- **锁定讯息也吃得到 PlaceholderAPI**：原本 `MenuItem.resolveLockMessage` 传 null plugin 给 `PlaceholderUtils.parse`，导致锁定按钮的 `deny_message` / `lock_message` 不会解析 `%placeholders%`；改为正确传递 `Plugin` 参数。
- **`JavaMenuListener` 单次评估**：点击选单按钮时原本 `isLocked()` 与 `getLockMessage()` 会各跑一次条件评估；新增 `LockState` record，一次评估同时取出 locked 状态与锁定讯息，避免重复计算且消除中间态不一致。

#### 解耦 / 可维护性
- **`RequirementService` 改吃 `Plugin` 接口**：建构式从 `RequirementService(MiaoMenu)` 改为 `RequirementService(Plugin)` + 测试友善的 `(Plugin, Server)` 双建构式；`evaluateAdvancement` / `getScore` 改用注入的 `Server` 而非 `plugin.getServer()`，方便单元测试。
- **`JavaMenu` 改吃 `Plugin` 接口**：把硬绑的 `MiaoMenu plugin` 字段替换为 `Plugin plugin`，降低耦合。
- **`MiaoMenu` 主类去掉 `final`**：plugin 主类无理由 final，去掉后便于将来测试与扩充。
- **`PlayerLifecycleListener` / `_Folia` onQuit null check**：保险起见对 `RateLimiter` 加 null 检查，避免 onDisable 期间竞态崩溃。
